# RunDetail 页面重构计划：迷你图例与月度热力图

## 1. 摘要 (Summary)
本计划旨在优化 `RunDetail` 页面的 UI 细节，将原本占据较大空间的“Aerobic Zones（有氧心率区间）”卡片重构为紧凑的迷你图例，并新增一个“月份迷你热力图”来可视化展示当前运动所在月份的运动频次，提升数据面板的信息密度与美观度。

## 2. 现状分析 (Current State Analysis)
- `RunDetailPanel` 当前通过 `grid-cols-5` 渲染了 5 个较大的卡片来展示有氧心率区间。
- 月度跑步距离（`monthlyDistanceKm`）仅以纯文本数字的形式居中展示，缺乏可视化反馈。
- `RunDetail.tsx` 中已经提取了该月的总距离，但尚未提取该月具体有运动的日期列表，无法直接供热力图使用。

## 3. 拟定修改 (Proposed Changes)

### 3.1 修改 `src/pages/RunDetail.tsx`
- **提取月度运动日期**：在现有的 `monthlyDistanceKm` 计算逻辑附近，新增一个 `monthRunDates` 的 `useMemo`。过滤出该月所有的跑步记录，并提取出 `YYYY-MM-DD` 格式的日期数组。
- **传递 Props**：将 `monthRunDates` 作为新属性传递给 `<RunDetailPanel />` 组件。

### 3.2 修改 `src/components/RunDetailPanel/index.tsx`
- **更新 Props 接口**：接收新传入的 `monthRunDates: string[]`。
- **重构 Aerobic Zones 迷你图例**：
  - 移除原有的 5 列网格卡片设计。
  - 使用 `flex` 布局创建一个连续的、高度较小（如 `h-1.5` 或 `h-2`）的圆角分段进度条。
  - 每个区间（Z1-Z5）占据相同的宽度，当前心率所在的区间保持 100% 不透明度，其他区间降低不透明度（如 20%）。
  - 在进度条下方添加紧凑的文本标签（Z1, Z2... Z5）。
- **新增月份迷你热力图 (Mini Monthly Heatmap)**：
  - 基于 `run.start_date_local` 解析出当前年份和月份，计算出该月的天数（28-31天）以及当月 1 号是星期几（以周一为起始日）。
  - 构建一个 `grid-cols-7 gap-[2px]` 的迷你网格。
  - 遍历渲染每天的格子（使用 `w-2.5 h-2.5` 或 `w-3 h-3` 尺寸）：
    - **无运动日期**：暗色背景（如 `bg-gray-800/50`）。
    - **有运动日期**：主题色背景（如 `bg-accent/60` 或 `bg-orange-500/60`）。
    - **当前查看的运动日期**：特殊高亮（如 `ring-1 ring-white bg-accent`）。
- **优化底部排版**：将迷你心率图例、月度跑步距离、以及迷你热力图进行组合排版。例如将月度距离和热力图包裹在一个带有背景色的圆角区域内（如 `bg-gray-900/50 p-4 rounded-lg`），左右两端对齐展示。

## 4. 假设与决策 (Assumptions & Decisions)
- **热力图起始日**：假设周一为每周的第一天，这符合绝大多数运动软件（如 Strava）的日历习惯。
- **热力图颜色逻辑**：为保持迷你热力图的简洁性，仅通过“是否有运动”来决定格子是否着色，而不根据当天的运动距离深浅渐变。
- **样式统一**：迷你热力图的着色将采用现有的 `bg-accent` (通常为橙红色) 或其他 Tailwind 色阶，以保持深色运动风的主题一致性。

## 5. 验证步骤 (Verification Steps)
1. 访问任意单次跑步的详情页（如 `/run/12345`）。
2. 确认底部的 Aerobic Zones 显示为紧凑的细条状迷你图例，且正确高亮了当前的心率区间。
3. 确认底部新增了月份日历网格，正确点亮了该月有跑步的日期。
4. 确认当前正在查看的跑步日期在热力图网格中具有明显的边框高亮。
