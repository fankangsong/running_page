# Tracks 页面日历方格重构计划

## 目标
将 `Tracks.tsx` 中当前 `RunningCharts` 区域（含两个 SVG 图片加载器）重构为“全年日历方格 + 轨迹可视化”组件：
- 移除现有 2 个图片加载器（年历图、网格图）
- 顶部显示周标题：`S M T W T F S`（周日到周六）
- 左侧显示月份英文缩写：`Jan. Feb. ... Dec.`
- 每日方格内部绘制当日跑步轨迹 `polyline`
- 按里程区间（`<=5`、`>5~10`、`>10~15`、`>15`）区分颜色
- 鼠标悬停方格时展示 Popover 详细跑步数据

## 实施步骤

### 1. 梳理并冻结替换范围
1. 锁定 `src/pages/Tracks.tsx` 中 `RunningCharts` 的挂载点（当前在第 174 行附近）。
2. 锁定 `src/components/RunningCharts.tsx` 为主要改造目标：保留 `year` 入参接口，替换内部渲染逻辑。
3. 明确不改动 `TracksStats`、筛选逻辑与 `displayRuns/displayYear` 过渡逻辑，确保改造只发生在图表区域。

### 2. 定义新组件结构（在 RunningCharts 内部或其子组件）
1. 建立三层结构：
   - 顶部：星期标题行（固定 7 列）
   - 左侧：月份标题列（12 个英文缩写）
   - 主体：按周排列的全年日历方格矩阵
2. 保持外层容器响应式与现有页面风格一致（Tailwind 体系），避免新增与现有主题冲突的全局样式。
3. 为无数据日期保留空方格占位，保证全年网格结构完整。

### 3. 构建全年日历映射数据
1. 基于当前 `year` 生成当年所有日期序列（1/1 到 12/31）。
2. 将日期映射为“周索引 + 星期索引（0=Sun）”，使列按周推进、行按周内天序排列。
3. 为每个月记录首日所在周，用于左侧月份标签对齐。
4. 将 `Activity[]` 按本地日期聚合为 `date -> runs[]`，并计算：
   - 当日总里程
   - 当日轨迹来源（优先第一条可用 `summary_polyline`，无则标记空）
   - 当日详细展示字段（次数、总里程、总时长、均速/配速基础值）

### 4. 实现 polyline 渲染策略
1. 在每个日期方格内部使用小型 `svg` 视窗绘制轨迹路径。
2. 对 `summary_polyline` 做解码并归一化到固定小视窗坐标（保留纵横比并居中）。
3. 按当日总里程设置线条颜色分段：
   - `<=5km`
   - `>5km 且 <=10km`
   - `>10km 且 <=15km`
   - `>15km`（高强度强调色）
4. 无轨迹时渲染浅色占位样式，保证视觉一致。

### 5. 实现 Hover Popover 明细
1. 复用项目现有 hover 交互模式（`group/absolute/opacity`）实现无侵入弹层。
2. Popover 展示字段至少包含：
   - 日期
   - 跑步次数
   - 当日总里程
   - 当日总时长
   - 代表性跑步名称（可选第一条）
3. 处理边界位置（靠近容器边缘时避免被裁切，可采用上下偏移或左右吸附策略）。

### 6. 替换旧加载逻辑并清理依赖
1. 删除 `RunningCharts` 中对两类 SVG 资源的动态加载流程（包括 loading 占位与 requestId 防抖竞态逻辑）。
2. 移除该组件内对 `assets/index.tsx` 与 `loadSvgComponent` 的依赖引用（仅限此次改造涉及的旧流程）。
3. 保持 `RunningCharts` 对外 API 不变（仍接收 `year`），保证 `Tracks.tsx` 无需额外改动或仅最小改动。

### 7. 样式与可用性校验
1. 校验周标题、月份标签、方格间距在常见屏宽下不重叠。
2. 校验空数据年份、部分数据年份、密集数据年份三类场景显示。
3. 校验 hover 行为在鼠标快速移动下不闪烁，不阻断页面滚动与点击。

### 8. 回归验证与交付
1. 编译级检查：确保 TypeScript 类型通过，无新增 lint/类型错误。
2. 功能回归：
   - 年份切换后日历与数据同步
   - 统计区与筛选区行为不受影响
3. 视觉回归：确认已不存在旧的两张图片式图表，仅保留新日历方格可视化。

## 交付结果
- `Tracks` 页面图表区从“静态 SVG 加载器”升级为“数据驱动的全年日历轨迹图”
- 实现基于里程区间的颜色编码与 Hover 明细交互
- 对外接口保持兼容，降低对其他页面模块的影响
