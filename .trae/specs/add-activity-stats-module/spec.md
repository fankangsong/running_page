# Activity Stats Module Spec

## Why
用户希望在热力图页面 (`Heatmap.tsx`) 中新增一个统计模块，以图表和详细指标面板的形式，支持按多维度（距离、时长、次数等）和多时间跨度（周、月、年、全部）进行灵活的数据聚合与查看，从而更直观、全面地掌握运动情况。

## What Changes
- **新增图表与统计组件**：在 `Heatmap.tsx` 页面内（如热力图下方）新增一个综合统计模块。
- **时间跨度切换**：提供“周”、“月”、“年”、“全部”的切换选项（Tab）。
- **时间段导航**：在不同时间跨度下，支持前后切换具体时间段（如通过前后箭头切换“2026年第16周”、“2026年第17周”或“1月”、“2月”等）。
- **指标维度切换**：支持通过右上角下拉菜单切换图表显示的 Y 轴数据维度（距离、时长、次数等）。
- **数据图表展示**：根据选中的时间跨度和维度，使用柱状图展示具体数据趋势（如按周展示周一到周日的数据分布）。
- **底部详细指标面板**：展示当前选中时间段内的汇总指标：总跑量（公里数）、总时长、总次数、平均配速、平均心率、最快配速、平均有氧区间。

## Impact
- Affected specs: `Heatmap.tsx` 页面布局与内容。
- Affected code: `src/pages/Heatmap.tsx` 及其子组件（新增 `src/components/ActivityStats` 等），可能会利用现有的数据处理逻辑。

## ADDED Requirements
### Requirement: 统计模块布局与交互
系统需要在热力图下方（或上方）提供一个功能完整的统计图表模块，布局参考所提供的视觉截图，但沿用项目现有 UI。
#### Scenario: 切换时间跨度与导航
- **WHEN** 用户点击“周”并点击前后箭头导航
- **THEN** 界面中间的标题更新为“XXXX年第X周”，图表 X 轴更新为周一至周日，图表数据及底部指标面板根据该周的真实活动数据重新计算和渲染。

#### Scenario: 切换指标维度
- **WHEN** 用户在右上角下拉菜单中选择“时长”
- **THEN** 图表重新渲染，Y 轴变更为时长单位，柱状图高度反映对应时间段内每一天（或月）的运动时长分布。

#### Scenario: 底部面板指标展示
- **WHEN** 页面加载或切换时间段时
- **THEN** 底部面板展示所有要求的指标：总跑量（公里数）、总时长、总次数、平均配速、平均心率、最快配速、平均有氧区间。缺失数据时展示默认占位符（如 0 或 -）。
