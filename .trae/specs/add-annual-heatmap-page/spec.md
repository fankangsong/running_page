# Annual Heatmap Page Spec

## Why

用户希望新增一个“年度热力图页面”，通过类似 GitHub 贡献图的形式（按年份聚合展示跑步/运动数据），直观地查阅全年的活跃情况。

## What Changes

* **新增页面**：在项目中新建热力图页面组件，并配置相应的路由以便访问。

* **页面布局**：页面包含标题、热力图主体区域、年份选择器三个核心模块。

* **年份选择器**：位于右上角，支持下拉或弹层形式，年份范围不少于近 10 年，切换时带无闪屏的平滑 loading 动画。

* **热力图渲染**：按年份展示 `[ { date: 'YYYY-MM-DD', value: number } ]` 格式的数据，前端需进行按年聚合；缺省数据渲染为灰色块。

* **热力图交互**：鼠标悬停显示单日详细数据，点击日块触发回调（预留接口），并具备良好的加载态与空态处理。

* **UI 规范统一**：页面色彩、字体、间距、圆角、交互状态等，必须沿用现有站点的设计体系。

* **响应式适配**：页面需适配桌面与移动端设备。

## Impact

* Affected specs: 页面导航/路由配置。

* Affected code: `src/pages` 新增页面文件，`src/components` 新增热力图与年份选择器组件，以及相关的路由配置文件。

## ADDED Requirements

### Requirement: 热力图页面基础布局与响应式

系统需要提供一个包含标题、热力图和年份选择器的响应式页面。

#### Scenario: 桌面端与移动端适配

* **WHEN** 用户在桌面端或移动端访问热力图页面

* **THEN** 桌面端完整展示全年网格；移动端通过自适应（如横向滚动或缩放）确保热力图信息可读，且标题与选择器排布合理。

### Requirement: 年份切换与平滑加载

系统需提供一个包含至少近 10 年选项的选择器，且在切换时数据区域有平滑加载态。

#### Scenario: 切换年份

* **WHEN** 用户通过右上角的选择器切换到另一个年份

* **THEN** 选择器收起，热力图区域出现平滑的 Loading 动画（避免整个页面闪烁），请求完成后渲染对应年份的数据。

### Requirement: 热力图渲染与交互

系统需接收特定格式的数据进行渲染，并支持 Hover 和 Click。

#### Scenario: 交互操作

* **WHEN** 鼠标悬停或点击某天的色块

* **THEN** 悬浮时通过 Tooltip 显示当天的日期和详细数据，无数据则展示灰块；点击时能触发组件向外暴露的回调函数。
